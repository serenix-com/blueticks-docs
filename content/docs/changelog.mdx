The API and each official SDK track versions independently.

## API specification

### v2.0.0 ‚Äî 2026-06-06 (breaking)

- **Pagination switched from cursor to offset.** Every list endpoint now takes
  `limit` + `skip` and returns `{ data[], limit, skip, total }`. The previous
  `cursor` query param and the `has_more` / `next_cursor` response fields are
  removed. `total` is the full count of matching items across all pages.
- **All responses now carry a `success` flag.** Success responses are
  `{ success: true, ... }` ‚Äî list endpoints add `data`/`limit`/`skip`/`total`,
  and single-resource / create / update responses nest the payload under
  `data` (`{ success: true, data: {...} }`). Errors are
  `{ success: false, error: { code, message, request_id } }`.

### v1.0.0 ‚Äî 2026-04-23

- Initial public release.
- Endpoints: `GET /v1/ping`, `GET /v1/account`.
- Error envelope contract locked (see [Errors & retries](/docs/errors)).

### v4.0.0 (Python ¬∑ Node) / v5.0.0 (PHP) / v2.0.0 (Ruby ¬∑ Go) ‚Äî 2026-05-27 (breaking)

- **Pagination switched from cursor to offset**, matching **API v2.0.0**:
  list calls now take `limit` + `skip` and return
  `{ data[], limit, skip, total }`. Replace any loop that read `has_more` /
  `next_cursor`.
- **`delete()` now returns a typed `{ id, deleted: true }` resource** instead
  of `void`.

### Ruby & Go SDKs ‚Äî 2026-05-01

- **Two new official SDKs joined the family:** Ruby (`gem install blueticks`)
  and Go (`go get github.com/serenix-com/blueticks-go`). Both expose the same
  resources as the other clients, in each language's idiomatic style ‚Äî Ruby with
  keyword args (`client.scheduled_messages.create(...)`), Go with `context.Context`
  and typed param structs (`client.Chats.SendMessage(ctx, ...)`). See the
  [Quickstart](/docs/quickstart) and any [API Reference](/docs/api) page for
  per-endpoint examples in all five languages.

### v3.1.0 ‚Äî 2026-04-25 (Node.js only)

- `BluetickError.details` field surfaces backend Zod validation issues
  with per-issue `{path, code, message}`. Existing `try/catch` code
  keeps working; the new field is additive.

### v3.0.0 ‚Äî 2026-04-25 (Python ¬∑ Node ¬∑ PHP)

- WhatsApp tools: `chats`, `groups`, `contacts`, `engines`, `utils`
  resources land alongside the existing `messages` / `audiences` /
  `campaigns` / `webhooks` set.

### v2.0.0 ‚Äî 2026-04-24 (Python ¬∑ Node ¬∑ PHP)

- Cursor pagination across all list endpoints. Replaces page-number
  pagination. Calls return `{data[], has_more, next_cursor}`; pass
  `cursor` back to fetch the next page.

### v1.0.0 ‚Äî 2026-04-23

- Initial public release of all three SDKs alongside the v1 API spec.

## MCP server

### v0.1.0 ‚Äî 2026-04-25

- Initial release of `@blueticks/mcp` ‚Äî local stdio MCP server wrapping
  the Node SDK. Exposes 9 tools, one per Blueticks resource
  (`scheduled_messages`, `chats`, `groups`, `audiences`, `campaigns`,
  `contacts`, `engine`, `webhooks`, `utils`). See [MCP server](/docs/mcp)
  for setup and the authoritative tool list.
